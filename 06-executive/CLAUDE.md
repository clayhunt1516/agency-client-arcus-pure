# 06 — Executive

> **For Claude:** Read this file before working in this folder.

## What this folder is

This folder is a **slot**. The universal core ships the folder and this orientation file; it does **not** ship executive report content.

Executive reporting synthesizes a client's operational departments — and those departments are industry-shaped. So executive report specifications are supplied by the **industry module**, not by the universal core.

## What you should find here

- **If an industry module has populated this folder:** executive report specs (and possibly SOPs) for Arcus Pure's vertical. Each report spec describes what the report contains and how it is produced.
- **If this folder holds only this file:** no industry-module executive content was deployed. That is the current state — Arcus Pure has no industry module. Executive reporting is not in scope for this deployment.

## Conventions

- Executive report specs follow the framework's report specification format.
- Executive reports synthesize operational and accounting data; they do not duplicate it. Source numbers come from QuickBooks Online and the operational systems of record.
